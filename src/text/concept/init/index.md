# 깃 저장소 생성
---
깃은 작성된 소스 코드 파일의 모든 변경 사항을 관리합니다.  

그렇다면 깃은 파일의 변경 내역을 어떻게 저장하고 관리할 수 있을까요?  
깃은 이러한 변경 사항을 전용 저장소(repository)(리포지터리)에 저장합니다.  

이 저장소는 일반적으로 사용하는 폴더와 유사하지만, 조금 차이가 있습니다.  
깃의 동작 방식을 이해하려면 저장소 동작 원리를 확실히 알아야 합니다.  

이제부터 하나씩 알아봅시다.
